## 📌 项目简介

**AuditArc 审迹** 是一个面向会计师事务所与企业内审部门的 AI 审计风险预警智能体系统。

传统审计长期依赖人工翻阅凭证、抽样核查、线下分析，不仅耗时耗力，更易因人员疲劳和经验差异导致风险识别滞后。AuditArc 以 AI 智能体重构审计风险识别流程，让审计从"人工找问题"变成"AI 主动告诉你问题在哪"。

---

## ✨ 核心功能

| 功能模块 | 说明 |
|--------|------|
| 🌐 多源数据集成 | 支持 Yahoo Finance（全球股票）、WRDS CSMAR（A股专业库）、演示数据三种接入方式 |
| 📰 舆情监控 | 实时抓取巨潮资讯、SEC EDGAR、Google News、新浪财经、东方财富等多源舆情信号 |
| ⚙️ 双模型并行识别 | 规则引擎（确定性路径）+ XGBoost AI 模型（学习型路径）双轨并行，结果互相印证 |
| 🛡️ 三重保险校验 | 规则阈值校验 → 业务合理性核查 → 多维度交叉验证，将误报率控制在 10%-20% |
| 🤖 AI 智能体推理 | 调用 Groq LLaMA 3.3 70B 大模型进行审计专家级语义推理，输出真实智能分析结论 |
| 📊 SHAP 可解释输出 | 每条风险结论自动生成特征贡献权重分解，彻底解决 AI 审计"黑盒"问题 |
| 🏭 多行业基准 | 支持制造业、零售业等不同行业基准，财务指标标准随行业切换自动变化 |
| 🔄 7×24 持续监控 | 系统自动唤醒，不间断完成数据采集、风险识别与预警推送 |
| 📄 一键导出 PDF | 生成包含完整证据链、AI 推理结论、准则引用的审计重点提示函 |

## 🚀 快速开始

### 环境要求

- Python 3.9+
- 网络可访问 Groq API（[groq.com](https://groq.com) 免费注册）

### 本地运行

```bash
# 1. 克隆仓库
git clone https://github.com/langwei528/audit-arc-v3.git
cd audit-arc-v3

# 2. 安装依赖
pip install -r requirements.txt

# 3. 配置 Groq API Key
# 打开 app.py，找到第 12 行，填入你的 Groq API Key：
# GROQ_API_KEY = "你的Key"

# 4. 启动应用
streamlit run app.py
```

### Streamlit Cloud 部署

1. Fork 本仓库到你的 GitHub 账号
2. 打开 [share.streamlit.io](https://share.streamlit.io)，点击 **New app**
3. 选择仓库、分支 `main`、入口文件 `app.py`
4. 点击 **Deploy**

## 🔍 风险识别逻辑

### 双模型并行

```
多源财务数据输入
        ↓
┌──────────────┐    ┌──────────────┐
│  规则引擎     │    │  XGBoost模型  │
│  刚性阈值校验  │    │  隐性组合异常  │
│  准则逻辑规则  │    │  行业对标评分  │
└──────┬───────┘    └──────┬───────┘
       └─────────┬──────────┘
                 ↓
           结果融合层
      一致→直接输出 / 冲突→触发第三重验证
                 ↓
           风险等级输出（极高/高/中/低）
```

### 三重保险校验

| 层级 | 机制 | 效果 |
|------|------|------|
| 第一重 | 规则阈值校验（宽口径，宁可多报） | 候选异常缩减约 40% |
| 第二重 | 业务合理性核查（季节性/扩产/政策排除） | 再缩减约 40% |
| 第三重 | 多维度交叉验证（2+ 独立维度同时异常） | 最终误报率 10%-20% |

### 核心识别风险类型

- 📈 **收入异常**：营收高增长但现金流背离，应收账款异动
- 💳 **应收账款异常**：周转率低于行业均值，赊销政策宽松
- 📦 **存货异常**：周转率下降，跌价风险，积压迹象
- 💰 **现金流异常**：经营现金流与利润严重背离
- 🏦 **负债率异常**：资产负债率超行业基准，偿债能力下降

## 🛡️ 数据安全与审计伦理

- **人机协同定位**：AI 仅提供风险线索与程序建议，最终审计定性与报告签发权始终保留在审计师手中
- **可解释性输出**：每条风险结论均附 SHAP 特征贡献权重，拒绝黑盒审计
- **准则依据**：每条风险自动引用对应的《中国注册会计师审计准则》条款
- **幻觉抑制**：AI 推理结论需基于量化指标数据，不接受无依据的主观推断
- **证据链完整**：PDF 报告包含原始数据、分析逻辑、准则引用、AI 推理四层证据
